## Topic 1: 网页端粘贴图片消失问题

**问题**：网页端（18789 OpenClaw webchat）粘贴图片后消失，图片没有传到小白 session

**排查结果**：
- OpenClaw 配置 `tools.media.image.enabled: true` ✅ 正常
- `image` 工具报错 `Unknown model: qwen/qwen-vl-plus` ——模型命名空间格式不对
- `agents.defaults.imageModel: "qwen/qwen-vl-plus"` vs DashScope API 模型名 `qwen-vl-plus` 不匹配
- 但图片在你本地浏览器能存住（刷新后消失，说明是浏览器端暂存，非服务端持久化）

**结论**：webchat 插件没有把粘贴的图片作为附件发送给 agent，图片只存在浏览器端小白看不到

**Workaround**（采用方案 B）：
- 桌面倒新截图 → 说「看图」→ 小白复制到 inbox → Python 脚本直调 DashScope API 读图
- 看图流程已调通，成功识别图片内容（红色 T 恤 + 黄色人脸剪影）

**待修**：`image` 工具的模型路由配置（`qwen/qwen-vl-plus` → 应改成 `qwen-vl-plus`），但老板选择暂不改，避免影响现有流程
