## 重大突破：新浪接口 + curl 并行（02:04）

### 根本问题
- baostock：login/logout 耗时 11秒/次，5482只 × ~1.1秒 ≈ 100分钟（无法在 exec 限制内完成）
- akshare：系统代理 10808 不通，Python requests 全部失败
- 代理：curl 直连可用，requests 走系统代理失败

### 解决方案
**新浪 K 线接口 + curl subprocess + Python ThreadPoolExecutor(50)**
- 接口：`https://money.finance.sina.com.cn/quotes_service/api/json_v2.php/CN_MarketData.getKLineData`
- 参数：`symbol=sh600000, scale=240, ma=no, datalen=5`
- 关键：curl subprocess 不走系统代理（Python requests 走代理会失败）
- CSV 格式：`sh.600000`（有点）→ 需转 `sh600000` 才能用于新浪接口

### 性能
- 5482 只股票 / 50 并行 = **59 秒**（史上最快！）
- 成功率：5475/5482 = **99.9%**
- 完全不受 exec 12 分钟限制影响

### 脚本路径
- `scripts/astock_sina_fetch.py` — 新浪接口 + curl 并行增量脚本
- 目标日期：`TODAY = "2026-04-16"`（当前硬编码）
