## parseEmbeddedJson 日志干扰 Bug 修复

**问题**：`openclaw memory status --json` 的 JSON 解析失败。输出里插件日志（如 `[plugins] [lcm] Ignoring...`）混在 JSON 前面，导致 `parseEmbeddedJson` 扫描到日志里的 `[` 字符而非真正的 JSON。

**根因**：
- `parseEmbeddedJson` 逐字符扫描找第一个 `[` 或 `{`
- 日志行 `[plugins] [lcm] Ignoring...` 里就有 `[` 字符
- 真正的 JSON 数组从第 4 行开始（同行只有单个 `[`）
- 原逻辑扫到的是日志里的括号，解析失败

**修复**：
- 文件：`openclaw-control-center/src/runtime/openclaw-cli-insights.ts`，`parseEmbeddedJson` 函数（约 line 574）
- 改为按行扫描，跳过以 `[` 或 `{` 开头但同行还有其他文字的行（`[plugins] ...`）
- 找到同行只有单个 `[` 或 `{` 的行作为 JSON 开始，再做 `JSON.parse`
- 编译 ✅ 服务重启 PID 64250 ✅

**验证**：老板刷新 Control UI，协作大厅和小红头像等记忆状态正常显示 ✅
