### 图片读取（解决 image tool 不支持智谱的问题）
- **根因**：OpenClaw 内置 image tool 的 pi-coding-agent ModelRegistry 不认智谱 zhipu provider，registry.find 返回 null
- **解决**：绕过 image tool，用 exec + curl 直调智谱 vision 接口
  ```
  图片 → base64 → curl POST https://open.bigmodel.cn/api/paas/v4/chat/completions
  → model=glm-4.6v-flashx → messages 含 image_url (data:image/jpeg;base64,...)
  ```
- Token 消耗极少（约 3080 tokens），成本低
- **记忆**：以后图片读取一律用此流程，不依赖 imageModel 配置

## 教训
- exec 命令要先汇报方案和步骤，征得同意后再执行
- 重启 gateway 必须先获得用户同意
- 修改文件前先备份

- 每次改动不要在原文件上 edit，容易出现嵌套 span 匹配问题；应该 write 完整文件
